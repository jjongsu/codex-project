# 트렌드 맞춤 게임 상세 기획서

## 공통 설계 원칙

- 첫 인상: 사이트 접속 후 5초 안에 시작 버튼과 조작 방식을 이해할 수 있어야 한다.
- 세션 길이:
  - 기본 3~7분
  - 실패 후 즉시 재시작 가능
- 조작:
  - 모바일 우선
  - `탭`, `드래그`, `스와이프` 중심
  - 데스크톱에서는 마우스와 키보드를 보조로 지원
- 랭킹:
  - 게임 오버 후 즉시 점수 제출
  - 게임별 Top N 랭킹
  - 이후 `오늘`, `주간`, `전체` 확장 가능
- 공통 HUD:
  - 현재 점수
  - 일시정지
  - 음소거
  - 재시작
  - 게임 오버 모달
  - 점수 제출 모달

## 1. `Block Jam Blitz`

### 한 줄 설명

- 8x8 보드에 블록을 배치해 줄을 지우고 콤보를 이어가는 모바일 우선 퍼즐 게임

### 목표 경험

- 규칙은 바로 이해되지만, 보드 운영 실력 차이가 점수로 크게 드러나야 한다.
- 한 판이 짧고, `이번엔 더 잘 둘 수 있었는데`라는 감각이 남아야 한다.

### 핵심 규칙

- 보드는 `8x8`
- 화면에는 항상 `다음 블록 3개`가 노출된다.
- 플레이어는 블록 하나를 보드에 배치한다.
- 가로 또는 세로 한 줄이 가득 차면 즉시 제거된다.
- 한 번의 배치로 여러 줄을 동시에 지울 수 있다.
- 배치 가능한 자리가 없으면 게임 오버다.

### 조작

- 모바일:
  - 블록을 드래그해서 보드에 놓는다.
- 데스크톱:
  - 마우스로 드래그 앤 드롭

### 점수 규칙

- 블록 배치 기본 점수
- 한 줄 제거 점수
- 동시 제거 보너스
- 연속 제거 콤보 보너스
- 보드 여유 칸 유지 보너스는 MVP 이후 검토

### 난이도 구조

- 초반에는 단순 도형 위주로 시작한다.
- 점수가 올라갈수록 긴 도형, 배치 제약이 큰 도형 비율을 높인다.
- 콤보가 끊기면 멀티플라이어가 초기화된다.

### MVP 범위

- 8x8 보드
- 블록 3개 큐
- 가로/세로 줄 제거
- 점수 계산
- 콤보 UI
- 게임 오버
- 점수 제출
- 랭킹 조회

### 2차 확장 아이디어

- `Breaker` 같은 구제용 스킬 1종
- 일일 시드 챌린지
- 미션형 목표
- 스킨 테마

### UI 요구사항

- 상단:
  - 점수
  - 일시정지
  - 음소거
- 하단:
  - 다음 블록 3개
  - 드래그 안내
- 게임 오버:
  - 최종 점수
  - 최고 기록 여부
  - 다시 시작
  - 점수 등록

### 서버 검증 힌트

- 비정상 점수 차단 기준으로 `세션 길이 대비 과도한 점수`, `짧은 시간 내 반복 제출`, `음수 또는 비정상 큰 수`를 우선 본다.
- 1차에서는 완전한 재연 검증보다 상한선 검증으로 시작한다.

## 2. `Snake Survivor`

### 한 줄 설명

- 스네이크의 방향 전환 재미에 생존형 웨이브와 강화 선택을 붙인 짧은 세션 액션 게임

### 목표 경험

- `Snake`를 아는 유저는 바로 이해하고, 처음 보는 유저도 조작이 단순해 금방 적응할 수 있어야 한다.
- 플레이가 길어질수록 단순 회피가 아니라 `퍼크 선택`과 `위험 관리`가 중요해져야 한다.

### 핵심 규칙

- 뱀은 자동으로 전진한다.
- 플레이어는 좌/우 방향 전환만 제어한다.
- 먹이를 먹으면 점수와 길이가 증가한다.
- 시간 경과에 따라 적 또는 위험 오브젝트가 늘어난다.
- 일정 구간마다 강화 선택이 나온다.
- 벽, 자기 꼬리, 적과 충돌하면 게임 오버다.

### 조작

- 모바일:
  - 좌/우 버튼 2개 또는 좌/우 스와이프
- 데스크톱:
  - 방향키 또는 `A` `D`

### 점수 규칙

- 생존 시간 점수
- 먹이 획득 점수
- 엘리트 적 처치 점수
- 연속 먹이 획득 보너스

### 적과 위험 요소 초안

- `Chaser`: 플레이어를 천천히 따라오는 적
- `Mine`: 일정 시간 후 활성화되는 고정 위험물
- `Spinner`: 회전 궤도로 움직이는 방해체

### 퍼크 초안

- `Shield`: 1회 충돌 무시
- `Magnet`: 먹이 흡입 범위 증가
- `Slow Time`: 짧은 시간 적 속도 감소
- `Score Boost`: 일정 시간 점수 배율 증가
- `Tail Burn`: 꼬리 일부를 태워 공간 확보

### MVP 범위

- 단일 맵 1종
- 먹이 시스템
- 적 2종
- 퍼크 4종
- 생존 시간 기반 난이도 증가
- 게임 오버
- 점수 제출
- 랭킹 조회

### 2차 확장 아이디어

- 보스 웨이브
- 일일 맵 변형
- 주간 시즌 랭킹
- 캐릭터 스킨

### UI 요구사항

- 상단:
  - 점수
  - 생존 시간
  - 현재 퍼크 아이콘
- 하단:
  - 모바일 조작 버튼
- 강화 선택 모달:
  - 3개 중 1개 선택
  - 선택 즉시 재개

### 서버 검증 힌트

- `생존 시간`과 `점수`의 비율 상한선을 둔다.
- 한 세션에서 가능한 최대 먹이 수와 적 처치 수를 기준으로 극단값을 제한한다.

## 3. `Brick Shot Rush`

### 한 줄 설명

- 각도를 조준해 볼을 발사하고, 내려오는 브릭을 지우며 최대한 오래 버티는 모바일 슈터형 퍼즐 게임

### 목표 경험

- 한 손 조작으로도 만족감이 높아야 한다.
- 반사각 계산이 어렵지 않지만, 고득점 구간에서는 숙련도가 확실히 드러나야 한다.

### 핵심 규칙

- 매 턴 플레이어는 발사 각도를 정한다.
- 볼이 모두 돌아오면 브릭이 한 줄 아래로 내려온다.
- 브릭의 숫자는 필요한 타격 횟수다.
- 브릭을 파괴하면 점수를 얻는다.
- 브릭이 위험 라인에 닿으면 게임 오버다.
- 추가 볼 또는 특수 아이템을 획득할 수 있다.

### 조작

- 모바일:
  - 드래그로 조준
  - 손을 떼면 발사
- 데스크톱:
  - 마우스로 조준 후 클릭/드래그 해제

### 점수 규칙

- 파괴한 브릭 체력 합산 점수
- 한 턴 다중 파괴 보너스
- 연속 클리어 콤보 보너스
- 오래 생존한 라운드 보너스

### 특수 오브젝트 초안

- `+1 Ball`: 다음 턴 볼 수 증가
- `Pierce`: 일정 수의 관통 효과
- `Split`: 충돌 시 볼 분열
- `Bomb Brick`: 주변 브릭에 광역 피해

### MVP 범위

- 단일 무한 모드
- 기본 브릭 생성
- 조준선 표시
- 추가 볼 픽업
- 특수 브릭 1종
- 게임 오버
- 점수 제출
- 랭킹 조회

### 2차 확장 아이디어

- 보스 브릭
- 라운드 이벤트
- 특별 미션 모드
- 일일 패턴 챌린지

### UI 요구사항

- 상단:
  - 점수
  - 현재 라운드
  - 보유 볼 수
- 조준 상태:
  - 예측 경로
  - 발사 취소 가능
- 게임 오버:
  - 위험 라인 도달 메시지
  - 최종 점수
  - 다시 시작
  - 점수 등록

### 서버 검증 힌트

- 라운드 수와 브릭 체력 총합 기준으로 가능한 최대 점수 범위를 계산한다.
- 비정상적으로 짧은 플레이 시간과 높은 점수 조합을 우선 차단한다.

## 구현 우선순위

1. `Block Jam Blitz`
2. `Snake Survivor`
3. `Brick Shot Rush`

## 공통 개발 메모

- 모든 게임은 `GameScene -> GameOver -> ScoreSubmit` 흐름을 공통화한다.
- 랭킹 UI, 결과 모달, 일시정지 모달은 공통 컴포넌트로 만든다.
- 게임별 점수 계산은 독립시키되, 제출 API는 하나로 통합한다.
- 차후 `오늘`, `주간`, `전체` 랭킹 필터를 붙일 수 있게 응답 구조를 너무 단순하게 고정하지 않는다.
